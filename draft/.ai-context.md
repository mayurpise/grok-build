## Synopsis

### Architecture in brief
Grok Build is a Rust monorepo (~80 workspace members, ~2200 `.rs` files) shipping the `grok` CLI/TUI and agent runtime. Composition root: `xai-grok-pager-bin` → interactive `xai-grok-pager` or shell modes (`run_headless` / `run_stdio_agent` / `run_leader`). Tools execute through `xai-grok-tools` with host effects in `xai-grok-workspace` (permission, FS, worktrees). Shared leaves live in `crates/common/` (tool protocol, compaction, computer hub). Vendored Mermaid SVG stack under `third_party/`. Graph engine: 87911 nodes / 580233 edges; package labels are coarse (`codegen`, `common`, …) and some are noise (`str`,`list`,`dict`,`int`).

type: cli
lang: Rust 1.92.0
pattern: Layered monorepo (bin → UI/runtime → tools/host → common)
build: cargo run -p xai-grok-pager-bin
test: cargo test -p <crate>
entry: crates/codegen/xai-grok-pager-bin/src/main.rs
config: user/project config via xai-grok-config + CLI flags + env

- [SEC] Permission order: PreToolUse hooks → deny/ask/allow rules → remembered grants → built-in read-only auto-approvals → mode policy; **deny wins** @ user-guide/22-permissions-and-safety.md

- [SEC] Mermaid/SVG path handles untrusted model output — vendored stack @ third_party/README.md
